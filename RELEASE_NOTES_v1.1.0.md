## Zenodo Integration

### Webhook Configuration
The repository is configured with a Zenodo webhook that automatically creates a new DOI version when a tag is pushed.

### Verification Steps
1. Push the v1.1.0 tag to origin (see above)
2. Visit https://zenodo.org/
3. Log in with repository credentials
4. Navigate to the QSOL UFF deposit
5. Verify the new version appears within 5-10 minutes

### Expected DOI
- **Concept DOI** (all versions): [10.5281/zenodo.17510648](https://doi.org/10.5281/zenodo.17510648)
- **v1.1.0 DOI**: Will be generated automatically by Zenodo
- **v1.0.0 DOI**: [10.5281/zenodo.17510648](https://doi.org/10.5281/zenodo.17510648)

### Webhook Trigger Details
- **Event**: Tag push matching pattern `v*`
- **Target**: GitHub Release or Git Tag
- **Action**: Zenodo creates new version with automatic DOI assignment

### Troubleshooting Zenodo Integration
If the webhook doesn't trigger:
1. Check GitHub webhook delivery logs:
   - Repository Settings → Webhooks → Zenodo webhook
   - Click "Recent Deliveries"
   - Verify response code 200

2. Manually trigger if needed:
   - Go to Zenodo deposit page
   - Click "New version"
   - Link to v1.1.0 release/tag

3. Verify webhook configuration:
   - URL: `https://zenodo.org/api/hooks/receivers/github/events/`
   - Content type: `application/json`
   - Events: Releases, Tags
